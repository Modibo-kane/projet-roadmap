Phase 1 Idéation & Validation 1–2 semaines
Objectif : transformer une idée brute en concept validé · Critère de sortie : le problème est réel, la cible est identifiée, l'idée tient face à la critique
1.1 — Définition du problème → Fiche problème
Formuler le problème en 1 phrase
Identifier les utilisateurs cibles
Analyser la concurrence existante
1.2 — Validation terrain → 5+ interviews, synthèse
Critique — ne pas coder avant cette étape
Conduire 5 à 10 interviews utilisateurs
Valider la volonté de payer
Phase 2 Cadrage & Architecture 1–3 semaines
Objectif : définir précisément quoi construire et comment · Critère de sortie : specs rédigées, stack choisie, environnements configurés
2.1 — Définition du MVP → Document de specs
Bloquant Tout le développement dépend de cette étape
Lister toutes les fonctionnalités imaginées
Rédiger les user stories
2.2 — Choix du stack technique → ADR (Architecture Decision Record)
Choisir le stack frontend / backend / base de données
Concevoir le schéma de données
2.3 — Setup de l'environnement → Dépôt git + CI/CD de base
Configurer le projet et les outils
Phase 3 Conception & Design 1–2 semaines
Objectif : avoir des maquettes validées avant d'écrire du code · Critère de sortie : maquettes haute-fidélité approuvées par au moins 2 utilisateurs test
3.1 — UX & Wireframes → Wireframes validés
Cartographier les parcours utilisateurs (user flows)
Créer les maquettes basse puis haute fidélité
3.2 — Design system de base → Charte UI, composants
Définir les fondations visuelles
Phase 4 Développement & Tests 4–12 semaines
Objectif : livrer le MVP fonctionnel et testé · Critère de sortie : toutes les user stories MVP passent les tests, app déployée en staging
4.1 — Setup dev & organisation → Backlog priorisé, conventions
Structurer le backlog et les sprints
4.2 — Développement backend → API fonctionnelle + tests
Construire l'API et la couche données
4.3 — Développement frontend → UI intégrée et fonctionnelle
Intégrer l'UI et connecter à l'API
4.4 — Qualité, sécurité & performance → Rapport de tests
Buffer Prévoir +30% de temps ici
Tests bout-en-bout et audit sécurité
Phase 5 Lancement 1–2 semaines
Objectif : mise en production et acquisition des premiers utilisateurs · Critère de sortie : app en prod, 100+ utilisateurs réels, métriques trackées
5.1 — Déploiement production → App live et stable
Bloquant Vérifier le runbook de déploiement avant go-live
Déployer et configurer l'infra de production
Gérer les aspects légaux et conformité
5.2 — Communication & acquisition → 100+ users, page de lancement
Lancer la communication
Phase 6 Post-lancement & Croissance Continu
Objectif : stabiliser, itérer et croître · Critère de sortie : rétention > 40% à J+30, feuille de route v2 définie
6.1 — Analyse & feedback → Rapport hebdo + backlog mis à jour
Instaurer une boucle de feedback continue
6.2 — Itération & V2 → Roadmap v2 priorisée
Planifier les prochains cycles
6.3 — Aspects admin, légaux & financiers → Entité créée, fiscalité gérée
Structurer l'entité juridique et financière

Risques critiques à anticiper

Construire sans valider → Mitigation : ne pas écrire de code avant 5+ interviews utilisateurs réels
Scope creep → Mitigation : geler les specs MVP et refuser toute feature avant le lancement
Dette technique précoce → Mitigation : prévoir 20% du temps de dev pour les tests et le refactoring
Sous-estimer le temps → Mitigation : multiplier toutes les estimations par 1.5 pour les phases 4 et 5
Négliger le légal → Mitigation : intégrer CGU / RGPD dès le début, pas après le lancement
